New API v2

Build with the File Analyzer API

Integração HTTP para validar ficheiros de forma segura. A API recebe PDF/JPG/JPEG, analisa o risco e devolve resultado pronto para usar em aplicações de escritório, backoffice e automações.

Base URL local: http://localhost:8000 · Produção: configurar domínio público via proxy/API gateway.

Endpoints

GET
/health

Confirma que a API está disponível.

POST
/scan?response_mode=file|json

Recebe ficheiro em multipart/form-data no campo file. Em JSON devolve estado, score de risco e links do job.

GET
/jobs/{job_id}/report

Descarrega o relatório JSON dessa análise.

GET
/jobs/{job_id}/download

Descarrega o ficheiro final produzido (aceite ou rejeitado).

Sem resultados para essa pesquisa.

Exemplos rápidos

# 1) Health
curl http://localhost:8000/health

# 2) Upload para obter resposta JSON
curl -X POST "http://localhost:8000/scan?response_mode=json" \
  -F "file=@/caminho/ficheiro.pdf"

# 3) Obter relatório do job
curl "http://localhost:8000/jobs/{job_id}/report"

Códigos de resposta

200Pedido concluído com sucesso.
413Ficheiro excede o limite permitido.
415Tipo de ficheiro não suportado.
422Ficheiro rejeitado pela análise.
500Erro interno durante processamento.
